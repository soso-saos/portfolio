Projet Cyber : Audit & Pentest Docker

Audit Black Box d'une infrastructure conteneurisée et exploitation de vulnérabilités.

1Contexte & Objectifs

Dans un environnement simulé reproduisant une infrastructure d'entreprise, l'objectif était de réaliser un audit "Black Box" (sans information préalable) sur un réseau Docker segmenté. Ce réseau exposait un service de partage de fichiers (Samba) et une application web interne.

2Le Défi Technique

Le challenge principal résidait dans le mouvement latéral (Pivoting). Une fois entré dans le réseau, il fallait :

  • Identifier et exploiter la faille SambaCry (CVE-2017-7494) pour obtenir un premier accès.
  • Mettre en place un tunneling pour atteindre les machines du réseau interne, invisibles depuis l'extérieur.
  • Réaliser une escalade de privilèges pour obtenir les droits root sur l'ensemble des conteneurs.

3Stack Technique & Outils

Metasploit Framework Nmap / Zenmap Python (Scripting Exploit) Wireshark (Analyse trafic) Docker

4Résultats & Livrables

L'audit a permis de compromettre intégralement l'infrastructure (accès Root validé sur les 2 machines cibles). J'ai produit un rapport technique détaillant chaque étape de la "Kill Chain" et proposant des correctifs précis (patching Samba, restriction des droits Sudo, durcissement des configurations Docker).
